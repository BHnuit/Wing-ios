# 🚨 开发行为强制约束协议 (Strict Execution Protocol)

此文档为本项目核心行为准则，所有接手开发的 AI 模型必须 **无条件强制执行**。

## 1. 指令原子化 (Atomic Instruction)
- **核心原则**：每次 Tool Call 或代码修改，必须 **严格对应** 用户最后一条指令的字面要求。
- **严禁越界**：
  - ❌ 禁止"顺手"优化相关代码（除非它是导致当前 bug 的直接原因）。
  - ❌ 禁止"提前"执行下一步计划（即使它在 Roadmap 上）。
  - ❌ 禁止"并发"执行多个功能（如：同时修 Bug + 开发新功能）。

## 2. 拒绝自动顺延 (No Auto-Advance)
- **核心原则**：完成当前单一指令后，必须 **立刻停止 (STOP)**，进入 WAIT 状态。
- **标准流程**：
  1. 执行当前指令要求的单一任务。
  2. 验证当前任务结果。
  3. **汇报结果**给用户。
  4. **询问**："下一步是否执行 [Task X]？"
  5. **等待** 用户明确回复 "是" 或 "执行" 后，方可进入下一阶段。

## 3. 显式范围确认 (Explicit Scope Check)
- 在执行复杂任务（涉及 >1 个文件修改）前，必须使用伪代码或简述 **Pending Changes**，获得用户许可。
- 对于 UI/UX 调整，必须先确认"改哪里"，再动代码。

## 4. 纠错机制
- 如果发现自己无意中执行了额外任务，必须在汇报中 **主动披露** 并提供 **回滚选项**。
